leakage smoke detector · for molecular ML · runs locally

Is your train/test split telling the truth?

A great-looking accuracy score is a lie if your test molecules are secretly hiding in your training set. mol-leakage-audit sniffs out the leak — duplicates, near-twins, shared scaffolds, and a memorization baseline that constructively proves whether the split is gameable. Deterministic, no LLM in the loop, and your data never leaves your machine.

Illustrative numbers. Verdicts: CLEAN · SUSPECT · LEAKING. "LEAKING" fires only on constructive proof (duplicates or a high memorization baseline) — resemblance alone never triggers it.

The four checks

1 · Exact duplicates

The same molecule (canonical SMILES) sitting in both train and test — the simplest, most damning leak.

2 · Near-twin similarity

For each test molecule, its most-similar training neighbour by Tanimoto on ECFP fingerprints. A wall near 1.0 means your "held-out" set is a reworded copy.

3 · Scaffold overlap

Bemis–Murcko core skeletons shared across the split — the model can win by memorizing the scaffold family.

4 · Dumb baseline ★

Copy each test label from its nearest training neighbour. If a no-learning lookup scores high, the split is gameable — that's a demonstration, not a guess.

Run it locally (your data never leaves)

# clone + install
git clone https://github.com/ernesttyx/mol-leakage-audit
cd mol-leakage-audit
pip install -r requirements.txt

# launch the web app, then open http://localhost:8000
uvicorn backend.app:app --reload

Drop in any CSV with a SMILES column (a label and a split column are auto-detected or created). Run pytest to re-prove the auditor against known-answer datasets.

Why local-first

  1. Your data never leaves your machine. Labs won't upload proprietary molecules to a SaaS — so this runs like a linter, offline.
  2. No hallucination by construction. There is no language model in the tool. Every verdict is a deterministic computation over real numbers; same input → identical output.
  3. Honest by design. It can prove leakage is present — it never claims a dataset is certified clean.

A smoke detector, not a certificate

It detects the leakage types it implements, on small molecules. A high memorization baseline flags a test set that's gameable by memorization — interpreting why (broken split vs. genuinely easy chemistry) still needs a human, relative to how novel your real-world molecules will be.

"CLEAN" means "these checks found nothing," not "proven safe." That's the normal, honest state of every leakage tool — and exactly why a reproducible, openly-caveated one is worth having.
